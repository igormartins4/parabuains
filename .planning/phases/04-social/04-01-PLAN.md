export type FriendshipStatus = 'pending' | 'accepted' | 'blocked';

export type FriendshipAction = 'accept' | 'reject' | 'block';

- ConflictError extends Error
- ForbiddenError extends Error

Interfaces necessarias (exportadas):
- IUserRepository: findByUsername, findById
- INotificationEmitter: emit(event, payload)

Classe FriendshipService com constructor(friendshipRepo, userRepo, notificationEmitter):

Metodo sendRequest(requesterId, targetUsername):
- Busca target por username, lanca NotFoundError se nao existe
- Lanca ForbiddenError se requesterId === target.id (auto-request)
- Chama findBilateral; lanca ConflictError se status pending ou accepted
- Lanca ForbiddenError se status blocked
- Cria o pedido e emite evento 'friendship.requested' via notificationEmitter

Metodo acceptRequest(userId, friendshipId):
- Valida que userId === friendship.addresseeId (lanca ForbiddenError caso contrario)
- Valida que status === 'pending' (lanca ConflictError caso contrario)
- Atualiza status para 'accepted' e emite 'friendship.accepted'

Metodo rejectRequest(userId, friendshipId):
- Valida que userId === addresseeId e status === 'pending'
- DELETA a row (nao apenas muda status)

Metodo blockUser(userId, friendshipId):
- Valida que userId e participante (requester ou addressee)
- Atualiza status para 'blocked'

Metodo removeFriendship(userId, friendshipId):
- Valida que userId e participante
- Para status 'pending': apenas requester pode cancelar
- Deleta a row

Metodo getMutualFriends(userAId, userBId):
- Busca amigos de A e amigos de B (via listFriends com limit=500)
- Retorna IDs presentes em ambas as listas

Funcao exportada: friendshipRoutes(fastify, options: { service: FriendshipService })

Mapeamento de erros de dominio:
- NotFoundError → 404
- ForbiddenError → 403
- ConflictError → 409
- Outros → throw (Fastify trata como 500)

Rotas a implementar:

POST /v1/friendships
- Body: { targetUsername: string }
- Schema JSON: required ['targetUsername'], minLength 1, maxLength 30
- preHandler: [fastify.authenticate]
- Chama service.sendRequest(request.user.id, body.targetUsername)
- Retorna 201 + friendship

PATCH /v1/friendships/:id
- Body: { action: 'accept' | 'reject' | 'block' }
- Schema: action enum ['accept', 'reject', 'block']
- preHandler: [fastify.authenticate]
- accept → service.acceptRequest, retorna 200
- reject → service.rejectRequest, retorna 204
- block → service.blockUser, retorna 200

DELETE /v1/friendships/:id
- preHandler: [fastify.authenticate]
- Chama service.removeFriendship(request.user.id, params.id)
- Retorna 204

GET /v1/users/me/friends
- Querystring: { cursor?: string; limit?: string }
- preHandler: [fastify.authenticate]
- Retorna { data: FriendshipWithUser[], nextCursor: string | null }
- nextCursor = ultimo username da lista ou null

GET /v1/users/me/friend-requests
- preHandler: [fastify.authenticate]
