# Plan 06-03: Birthday Scheduler

## Objective

Implement the timezone-aware birthday reminder scheduler: a BullMQ daily cron job that queries upcoming birthdays, checks idempotency, and enqueues reminder jobs. Add a birthday reminder consumer that sends email + push. Add notification preferences settings UI at `/settings/notifications`.

### Task 1 (auto): BirthdaySchedulerRepository

Create `apps/api/src/modules/notifications/birthday-scheduler.repository.ts`:

```typescript
export interface BirthdayTarget {
  userId: string;         // friend's userId (the birthday person)
  username: string;
  displayName: string;
  email: string;          // birthday person's email
  birthDate: string;      // "YYYY-MM-DD"
  timezone: string;       // friend's timezone
  notificationSendHour: number;
  friendIds: string[];    // list of friend userIds who should be notified
}

export interface RecipientInfo {
  userId: string;
  email: string;
  displayName: string;
  timezone: string;
}

export interface IbirthdaySchedulerRepository {
  // Get all users with birthdays in the next N days (across all timezones)
  // Returns pairs: (recipient who should get notified, birthday person)
  getUpcomingBirthdayPairs(daysAhead: number): Promise<Array<{
    recipient: RecipientInfo;
    birthdayPerson: { userId: string; username: string; displayName: string; birthDate: string };
  }>>;
}
```

Query: join `users` (birthday persons) with `friendships` (accepted) to get all (recipient, birthdayPerson) pairs where the birthday person has a birthday within `daysAhead` days from NOW (in UTC, conservative window). Use `extract(month from birth_date)` and `extract(day from birth_date)`.

### Task 2 (auto): Birthday scheduler cron job


The cron job uses BullMQ's repeatable job feature. On startup, add a repeatable job to the `notifications` queue:

```typescript
await getNotificationsQueue().add(
  'birthday.scan',
  {},
  {
    repeat: { pattern: '0 0 * * *' }, // midnight UTC daily
    jobId: 'birthday-daily-scan',
  }
);
```

The worker processes `birthday.scan` jobs:

1. Call `repo.getUpcomingBirthdayPairs(8)` (8-day window covers all timezones for 7-day reminders)
2. For each pair, determine which reminder type applies: `D7`, `D3`, `D1`, `Dday`
   - Use Luxon to calculate `daysUntil` in the **recipient's** timezone
3. Check idempotency: `notificationLogRepo.existsForYear(recipientId, birthdayPersonId, 'email', reminderType, currentYear)`
4. If not sent: enqueue `birthday.reminder` job with all needed context

```typescript
// birthday.reminder job payload
{
  type: 'birthday.reminder';
  recipientId: string;
  birthdayPersonId: string;
  birthdayPersonName: string;
  birthdayPersonUsername: string;
  daysUntil: number;
  birthMonthDay: string; // "MM-DD"
  reminderType: 'D7' | 'D3' | 'D1' | 'Dday';
}
```

**CRITICAL**: `daysUntil` mapping:
- 7 days → reminderType `D7`
- 3 days → reminderType `D3`  
- 1 day → reminderType `D1`
- 0 days → reminderType `Dday`
- Others: skip (not a configured reminder day)
